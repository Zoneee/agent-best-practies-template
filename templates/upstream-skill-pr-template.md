# 上游回流 PR 模板

> 使用说明：当你需要将下游项目中经过验证的 Skill 改动回流到模板主仓时，请使用本模板填写 PR 正文。  
> 本模板用于发起"受控回流 PR 提案"，不是自动同步机制。所有回流内容需经过人工审阅后方可合并。

## 为什么需要这次回流

（说明这次改动为何具有通用价值，而不仅仅是当前项目的特殊需求。建议说明：）

- 是否在多个任务或多个项目场景中都出现过类似需求
- 为什么现有模板 Skill 不足以覆盖这个场景

## 审阅检查清单

**发起方自查（回流前必须完成）：**
- [ ] 所有项目专属内容已剥离
- [ ] 改动经过实际任务验证，有可引用证据
- [ ] 已明确回流目标（base / overlay）
- [ ] 已填写非回流内容表格
- [ ] 已评估对其他项目的兼容性影响
- [ ] PR 正文符合本模板格式

**评审方检查（模板主仓维护者）：**
- [ ] 变更摘要清晰，目的明确
- [ ] 回流目标选择合理
- [ ] 验证证据可信
- [ ] 非回流内容已妥善处理
- [ ] 无项目专属信息泄露风险
- [ ] 与现有 Skills 体系兼容
