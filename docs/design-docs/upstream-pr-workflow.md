# 设计说明：下游 Skill 改动回流模板主仓工作流

> 状态：有效  
> 类型：工作流设计说明  
> 负责人：模板维护团队  
> 创建日期：2026-03-25

---

## 问题背景

随着 `agent-best-practies-template` 被多个下游项目采用，各下游项目在实践中会产生对模板 Skills 的改进和补充。这些改进往往源于真实任务中的验证经验，具有潜在的通用价值。

然而，目前没有一套标准化机制来处理"下游改动如何回流到模板主仓"这个问题，导致：
- 有价值的共性经验散落在各个下游项目中，无法沉淀到模板主仓
- 下游项目不清楚哪些内容适合回流、回流时应剥离哪些内容
- 缺乏统一的 PR 格式，维护者难以评估回流内容的质量和适用范围
- 存在把项目专属内容误回流到模板主仓的风险

---

## 目标

1. 为下游项目提供标准化判断框架：某个 Skill 改动是否适合回流？
2. 提供回流目标选择指导：应进入 `base` 还是特定 `overlay`？
3. 提供 PR 正文模板：确保回流提案格式统一、内容完整
4. 提供辅助脚本设计：未来可通过脚本自动化生成候选文件和提案草稿
5. 通过"受控 PR 工作流"而非自动同步机制保证回流质量

---

## 非目标

以下内容明确不在本工作流的范围内：

- **自动向上游仓库发起 PR**：本工作流不做自动跨仓库写操作
- **自动反向同步**：不是"下游改了什么，模板主仓就自动跟着改"
- **全量文档回流**：仅针对 Skills 文件，不支持业务文档、架构文档、运行手册等
- **实时同步监控**：不做持续监听下游改动的机制
- **强制推送**：模板主仓维护者对所有回流 PR 有完全的接受或拒绝权

---

## 方案原则

### 1. 受控优于自动
所有回流必须经过人工发起和人工审批，不存在自动写入机制。这是为了：
- 确保项目专属内容被妥善剥离
- 确保回流内容经过语义判断，具有通用价值
- 保持模板主仓变更历史的可读性和可追溯性

### 2. 剥离优先于回流
在提交回流 PR 前，必须先完成内容剥离：去除所有项目专属名称、内部系统引用、业务特定约束。剥离后的内容才是候选回流内容。

### 3. 验证先于提议
只有在实际任务中经过验证的改动，才适合提议回流。未经验证的想法或草稿不应进入回流流程。

### 4. 分层回流
根据改动的适用范围，分别回流到 `base`（全项目通用）或特定 `overlay`（某类项目通用），而不是简单地"回流到主仓"。

### 5. 最小改动
回流 PR 应尽量聚焦，每次只包含一个 Skill 的改动，避免大范围混合。这与模板主仓本身的 `small-safe-prs` 原则保持一致。

## 与前面模板同步体系的关系

本工作流是多项目模板同步体系中"双向流动"的一部分，与其他机制的关系如下：

```text
模板主仓（上游）
  ↓ 正向同步（由 sync-template.sh 驱动）
下游项目
  ↓ 实践验证、改进积累
下游项目
  ↓ 受控回流（由本工作流驱动）
模板主仓（上游）
```

### 与正向同步（sync-template.sh）的关系
- **正向同步**：模板主仓 → 下游项目，由 `tools/agent-template/sync-template.sh` 驱动
- **受控回流**：下游项目 → 模板主仓，由本工作流（`skill-upstream-pr` + `propose-skill-upstream-pr.sh`）驱动
- 两者互补，共同构成一个可持续演进的知识闭环

### 与 skills/index.md 的关系
新增或修改的 Skill 被合并到模板主仓后，需同步更新 `skills/index.md`，以便后续正向同步时下游项目能正确使用新 Skill。

### 与 CHANGELOG.md 的关系
每次回流 PR 合并后，建议在 `CHANGELOG.md` 中记录变更，说明回流来源和改动摘要，便于下游项目了解版本差异。

## 风险与后续事项

### 已知风险
- **内容剥离遗漏**：手动剥离时可能遗漏项目专属内容，建议评审方重点检查
- **回流目标误判**：下游项目可能对 `base` vs `overlay` 的边界判断不准确，维护者应在 PR 评审时明确指出

### 后续事项
- [ ] 实现 `propose-skill-upstream-pr.sh` 脚本（参考 `tools/scripts/propose-skill-upstream-pr-design.md`）
- [ ] 在模板主仓 README 中补充"如何向本仓库贡献 Skill"章节
- [ ] 制定回流 PR 的 SLA（评审响应时间目标）
